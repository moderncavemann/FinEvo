# No-visible-cue control

This variant keeps the endogenous sentiment state, its macro update, and its
numeric contribution to episodic state-similarity retrieval active, but
suppresses the explicit sentiment index/label generated by
`MarketSentiment.get_sentiment_prompt()` from the LLM decision prompt. The
legacy retrieval trace's separate `regime_match` component remains fixed at
0.0, so this control must not be described as preserving a working
"regime-conditioned retrieval" mechanism.

The same-day paired variants are `E9/visible-cue-control` and
`E9/no-visible-cue`. Both use GPT-5.2, 10 agents, 24 months, and the standard
five seeds. They differ only in `show_regime_cue`.

## Validation

Run the code-path and matrix assertions without making model calls:

```bash
python artifacts/no_visible_cue/validate_no_visible_cue.py
python run_emnlp_experiments.py \
  --exps E9 \
  --models gpt52 \
  --seeds 13,21,42,87,2026 \
  --workers 1
```

The validator checks that the experiment remains full FinEvo, uses the intended
10-agent/24-month dimensions, passes the paired `--show_regime_cue True/False`
flags, and does not call the cue-text renderer on the hidden path. Its
two-agent/one-month simulator smoke also checks that, after removing the cue
sentence, the paired prompts and internal numeric retrieval states are equal.

## Formal run status

The same-day five-seed pair completed on 2026-07-10: 10 agents, 24 months,
seeds 13, 21, 42, 87, and 2026. All 10 runs have zero invalid-action and API
error rates. Normalized outputs are under:

- `runs/E9/GPT-5.2/finevo/visible-cue-control/seed_<seed>/`
- `runs/E9/GPT-5.2/finevo/no-visible-cue/seed_<seed>/`

The paired configuration audit passed for all five seed pairs. Results and
conservative inference are in `paired_report.md`, with machine-readable rows in
`paired_metrics_by_seed.csv`, `paired_metrics_summary.csv`, and
`paired_analysis_status.json`.

To reproduce the formal runs:

```bash
python run_emnlp_experiments.py --run \
  --exps E9 \
  --models gpt52 \
  --seeds 13,21,42,87,2026 \
  --workers 8
```

Then regenerate the strict paired analysis:

```bash
python artifacts/no_visible_cue/analyze_paired_results.py
```

It validates same-day matched configurations and compares only the two E9
variants; it never substitutes old E2 rows.

SciPy is optional for descriptive summaries but required for the reported
paired Student-t confidence intervals and p-values. The exact SciPy version
used is recorded in `paired_analysis_status.json`.

The formal runs were executed from a dirty worktree whose base commit predates
the E9 flag. Exact source and patch hashes are therefore recorded in
`code_provenance.json`; the `code_commit` field alone is not sufficient to
reconstruct these runs.

The validator itself uses a deterministic stub model and makes no external API
call; it is separate from the completed formal GPT-5.2 runs.

An external GPT-5.2 smoke test was also completed for both variants with the
same seed, 10 agents, and one month. See `smoke_report.md`; it is a code-path
check only, not an experimental result.
